# 🐾 小怪兽日记 (Monster Story Manager)

该readme仅作展示最新功能，以及作者实现的思路，为方便查看，使用了ai修缮文本格式，但项目内部并无使用，作者认为在学习阶段使用ai无法吸收知识，任务纯手搓

---

## 📖 前言故事与任务目标

在一个遥远的代码星球上，住着一群可爱的小怪兽。它们每天都有自己的小故事，比如 *"今天学了喷火"*、*"收集了3颗宝石"*、*"飞行撞到树"* 等。但是最近小怪兽们有点混乱，有的忘记了自己的故事，有的把故事记混了...它们急需一个 **“故事管理大师”** 来帮助记录日常故事！

---

## ✨ 项目功能

在完成了基本要求的基础上，加了一点小小的优化：

### 💡 1. 个人扩展与优化s
- **自定义异常处理：** 写了一个 `StoryNotRightException`，在添加故事的时候，如果不小心输入了空的标题或内容，可以提示重新输入。
- **面向对象封装：** 没用 String 数组存故事，而是用了 `ArrayList<Story>`。让每个 `Monster` 自己管理自己的故事对象，感觉这样更符合逻辑，并且这样就可以存复数个故事，也避免了长篇大论不好存的弊端
- **预留了文件保存功能：** 在 `Story` 类里面加了 `java.io.File` 对象 (`new File("day1/storys")`)，想着以后也许能把故事真的存到电脑的 txt 文件里。
- **故事分类：** 增加了一个 `type` 属性，可以把故事分成冒险、开心、伤心等类型，完成了扩展任务里的一个小要求。
- **Switch新写法：** 菜单部分用了 Java 14+ 的 `switch (choice) ->` 写法，不用再写一堆 `break` 了，看起来很清爽。

### 👾 2. 基础功能
- **小怪兽管理：** 可以在控制台输入名字和年龄创建小怪兽，还能查看所有登记过的怪兽。
- **故事管理：** 选中一个小怪兽后，能给它写新的故事。输入名字就能打印出这个小怪兽所有的故事列表。
- **简单统计：** 算一算现在图书馆一共有多少个故事。

## 🏗️ 类设计思路

### 🧩 `Monster` 类
- **主要属性：**
  - `name`: 名字
  - `age`: 年龄
  - `stories`: 用一个 `ArrayList` 存这只怪兽自己的故事
- **主要方法：**
  - `addStory(Story story)`: 把写好的故事装进来，里面顺便检查一下属不属性不合规。
  - `printStoriesTitle()`: 返回这只怪兽所有故事的名字。

### 📕 `Story` 类
把故事单独做成一个类，以后想加什么属性比较方便：
- **主要属性：** 
  - `type` (故事类型)
  - `title` (故事标题)
  - `storys` (File对象，预留着用来存大段文字)
