## 1. 关键逻辑与开发思路

### 关于 ID 的处理 (构造方法)
在执行 `add` 操作时，数据库的 ID 通常是自增的，但我们在 Java 里封装对象时，有时需要手动传入 ID 进行关联，或者在查询后拿回 ID。
> **思路：** 专门再开一个带 ID 的构造方法。
> * 默认构造：用于新录入（ID 由数据库生成）。
> * 带参构造：用于从数据库查出来后，完整还原学生/成绩的所有信息。

### 防止时间被“默认值”覆盖 (核心心法)
这是项目里最绕的一个点。我们在 `Score` 类里初始化 `exam_time` 时，默认给的是 `new Date()`（即录入时的当前时间）。
> **问题：** 当我们从数据库执行 `query` 查询时，`rs` 结果集里的数据是 new 出来的。如果不处理，它会直接显示 new 出来的“今天”，而不是数据库里存的“考试那天”。
> **解决：** 在 `while(rs.next())` 搬运数据时，必须调用 `setExam_time(rs.getTimestamp(...))`。这一步是为了强制覆盖掉 Java 对象默认生成的时间，保住数据库里的真实考试时间。



### 模糊查询：拼字符串的艺术
模糊查询其实没那么神，本质上就是在 SQL 语句里“玩拼图”
> **思路：** > 1. SQL 语句保持干净：`WHERE name LIKE ?`。
> 2. 脏活累活交给 Java 拼：在 `stmt.setString` 时，手动加上两个百分号 `"%"+name+"%"`。
> 3. 这样不管用户搜“张”还是“张三”，都能准确匹配到。

## 2. 数据库设计思路

* **多表关联**：查询成绩时不能光看 `score` 表，必须带着 `student` 表玩（用 `student.id = score.student_id` 关联），不然你只知道分，不知道是谁考的。
* **级联删除**：删学生前，必须先带走他的所有成绩。我用了 `setAutoCommit(false)` 开启事务。万一成绩删了但学生没删掉，直接 `rollback` 回滚，保证数据不丢、不乱。

## 3. 功能清单 (Main 菜单)

运行 `Main.java` 后，你可以通过菜单进行以下操作：
1. **录入成绩**：为指定 ID 的学生增加科目和分数。
2. **删除学生**：一键清理学生及其所有关联成绩（事务保护）。
3. **按 ID 查询**：看某个具体学生的成绩单。
4. **模糊查询**：输入名字关键词，跨表抓取数据。
